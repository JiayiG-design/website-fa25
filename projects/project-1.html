Mobile Banking App Redesign

UX Design • 2024 • Lead Designer

A comprehensive redesign of a mobile banking application focusing on accessibility, user trust, and intuitive navigation for diverse user groups across different age ranges.

Challenge

The existing mobile banking app had significant usability issues, particularly for older users and those with accessibility needs. Users reported difficulty navigating the interface, understanding financial information, and completing basic transactions. The app also lacked modern design patterns and failed to build trust through its visual design.

Key challenges included: complex navigation structure, poor information hierarchy, lack of accessibility features, outdated visual design, and insufficient error handling and feedback mechanisms.

Research & Discovery

I conducted extensive user research including interviews with 25 users across different age groups, usability testing sessions, and competitive analysis of leading banking apps. The research revealed that users valued simplicity, clear communication, and a sense of security above all else.

Design Process

The design process followed a human-centered approach, starting with user journey mapping and information architecture restructuring. I created wireframes and prototypes that were tested iteratively with users to ensure the design met their needs and expectations.

Key Design Solutions

Simplified Navigation: Reduced the main navigation to 4 core functions (Accounts, Transfer, Pay, More) with clear visual hierarchy and intuitive icons.

Enhanced Accessibility: Implemented high contrast modes, larger touch targets, voice-over compatibility, and clear typography with sufficient spacing.

Trust-Building Design: Used a clean, professional color palette with subtle gradients and shadows to create a sense of security and reliability.

Progressive Disclosure: Organized complex information in digestible chunks with clear calls-to-action and contextual help.

Results

The redesigned app showed significant improvements in user satisfaction and task completion rates. Usability testing revealed a 40% reduction in task completion time and a 60% increase in user confidence when performing financial transactions.

Key metrics improved:

  • Task completion rate: 85% → 96%
  • User satisfaction score: 3.2/5 → 4.6/5
  • Accessibility compliance: 60% → 95%
  • Support ticket reduction: 35%

Reflection

This project reinforced the importance of inclusive design and user research in financial applications. The most valuable insight was understanding that trust in banking apps is built not just through security features, but through clear communication and predictable interactions.

The iterative testing process was crucial in identifying pain points that weren't apparent in initial user interviews. Moving forward, I would incorporate more diverse user groups in the testing phase and consider the emotional aspects of financial management more deeply in the design process.
